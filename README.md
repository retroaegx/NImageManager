# NovelAI Image Manager

## 起動
- Windows: `run.bat`
- macOS/Linux: `sh run.sh`

起動後にコンソールへ URL が表示されます。
- Local: http://localhost:32287
- Tunnel: https://...trycloudflare.com (有効な場合)
## 公開用アドレスを固定する（Quick Tunnel 以外で運用したい場合）

デフォルトだと Quick Tunnel で公開用アドレスを用意してくれますが、起動時に毎回アドレスが変わります。  
Cloudflare を固定接続、ドメイン移管することでそちらで接続も可能です。（下記の内容は無料のDDNSを活用してアドレスを固定する例です）

### 1. DDNS Now（ddns.kuku.lu）のトップへ行きドメインを登録する
左メニューの新規登録で

- 希望サブドメイン：****（英数と `-` など）
- パスワード：6文字以上

### 2. Cloudflare にドメイン移管する
- Cloudflareのトップ（https://dash.cloudflare.com/login ）へ行きログイン（必要なら登録）する。
- ログイン後、ダッシュボード（左上のアドレスをクリック）> ドメインを選択し、右ペインから「+ドメインのオンボード」を選択
- 「既存のドメインを入力」に登録したアドレスを入力し、「続行」を押下する。
- プラン選択画面で Free を選択する。
- 「DNS レコードを確認する」画面で「アクティベーションを続行」を押下する。
- 「現在のネームサーバーを Cloudflare ネームサーバーに置き換える」の欄に表示されているDNSネームサーバーのアドレス2つをメモする。
- DDNS Now の設定画面（https://ddns.kuku.lu/control.php ）を開く。
- NSレコード欄のすぐ下にある「認証する」ボタンから電話認証してください。自身の電話番号入力して、進んだ先に表示される電話番号に自分の電話番号から発信、3秒後切断するだけです。
- 再び DDNS Now の設定画面（https://ddns.kuku.lu/control.php ）を開く。
- A/AAAA/CNAME/サブドメイン/ワイルドカードを全部空にし、下のほうにある「設定を更新する」ボタンを押下する。
- NSレコード欄に先ほどメモした2つのアドレスを入力（改行して入力すること）する。
- 下のほうにある「設定を更新する」ボタンを押下します。再び設定画面にリダイレクトされれば完了です。

### 3. パソコンのトンネル接続恒常化
- Cloudflareのトップ（https://dash.cloudflare.com/login ）へ行きログインする。
- Cloudflareのドメイン > 右ペインのドメイン名 > 画面右側の「DNS レコード」を開きます。`nim-sample.f5.si` の DNS 管理の中にチェックボックスのあるレコードが"あれば"、チェックを入れてレコードの削除を行ってください。
- ログイン後、ダッシュボード（左上のアドレスをクリック）> ネットワーク > Tunnels を選択し、右ペインから「トンネルを作成」ボタンを押下
- 任意のトンネル名を入力し、「トンネルを作成」を押下する。
- インストールと実行欄に表示されている手順に従い作業してください。一番下の「接続状態」に「トンネルが正常に接続されました」と表示されれば完了です。表示される msi をインストールし、コマンドプロンプトを管理者権限で実行、表示されているコマンドを実行するだけです。Ubuntuで環境構築したい場合は Debian の手順で実行してください。完了したら「続行」ボタンを押下してください。
- ダッシュボード（左上のアドレスをクリック）> ネットワーク > Tunnels を選択し、右ペインから作成したトンネル名を選択してください。
- 画面タブの Routes 欄の中にある「+ Add route」ボタンを押下
- 「Published application」を押下
- 表示されたフォームで右上の「Domain」に登録したドメインが表示されていることを確認し、「Service URL*」の欄に `http://127.0.0.1:32287` を入力し、「Add Route」ボタンを押下する。

あとは登録したアドレス（`https://xxxx.f5.si`）にブラウザでアクセスしてみてください。  
`.env` の `NAI_IM_PUBLIC_BASE_URL` の `=` の頭の `#` を削除し、`=` の後ろに上のアドレスを記載してください。起動時にそのアドレスを表示してくれます。  
アカウントの作成やパスワードリセット時にこの値を使用するため間違えないように必ず記載してください。  

## 初期ログイン
起動後に アカウント名とパスワードを入力して作成します。  
マスターアカウント扱いでこのアカウントでしか出来ないことが複数あるので、パスワードを忘れないように気を付けてください。  

## データ保存場所
DB/画像/派生: `server/data/`  
初期化したい場合はこの中のファイル群をすべて削除してください。  
オリジナルファイルもここに保存されます。
