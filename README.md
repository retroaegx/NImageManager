# NovelAI Image Manager

## 起動
- Windows: `run.bat`
- macOS/Linux: `sh run.sh`

起動後にコンソールへ URL が表示されます。
- Local: `http://localhost:32287`
- Tunnel: https://...trycloudflare.com (有効な場合)
## 公開用アドレスを固定する（Quick Tunnel 以外で運用したい場合）

デフォルトだと Quick Tunnel で公開用アドレスを用意してくれますが、起動時に毎回アドレスが変わります。  
Cloudflare を固定接続、ドメイン移管することでそちらで接続も可能です。（下記の内容は無料のDDNSを活用してアドレスを固定する例です）

### 1. DDNS Now（ddns.kuku.lu）のトップへ行きドメインを登録する
[DDNS Now](ddns.kuku.lu)へアクセスし、左メニューの新規登録で

- 希望サブドメイン：****（英数と `-` など）
- パスワード：6文字以上

で登録します。  

### 2. Cloudflare にドメイン移管する
- [Cloudflareのトップ](https://dash.cloudflare.com/login)へ行きログイン（必要なら登録）する。
- ログイン後、ダッシュボード（左上のアドレスをクリック）> ドメインを選択し、右ペインから「+ドメインのオンボード」を選択
- 「既存のドメインを入力」に登録したアドレスを入力し、「続行」を押下する。
- プラン選択画面で Free を選択する。
- 「DNS レコードを確認する」画面で「アクティベーションを続行」を押下する。
- 「現在のネームサーバーを Cloudflare ネームサーバーに置き換える」の欄に表示されているDNSネームサーバーのアドレス2つをメモする。
- [DDNS Now の設定画面](https://ddns.kuku.lu/control.php)を開く。
- NSレコード欄のすぐ下にある「認証する」ボタンから電話認証してください。自身の電話番号入力して、進んだ先に表示される電話番号に自分の電話番号から発信、3秒後切断するだけです。
- 再び [DDNS Now の設定画面](https://ddns.kuku.lu/control.php)を開く。
- A/AAAA/CNAME/サブドメイン/ワイルドカードを全部空にし、下のほうにある「設定を更新する」ボタンを押下する。
- NSレコード欄に先ほどメモした2つのアドレスを入力（改行して入力すること）し、下のほうにある「設定を更新する」ボタンを押下する。
  
再び設定画面にリダイレクトされれば完了です。

### 3. パソコンのトンネル接続恒常化
- [Cloudflareのトップ](https://dash.cloudflare.com/login)へ行きログインする。
- Cloudflareのドメイン > 右ペインのドメイン名 > 画面右側の「DNS レコード」を開きます。`nim-sample.f5.si` の DNS 管理の中にチェックボックスのあるレコードが"あれば"、チェックを入れてレコードの削除を行ってください。
- ログイン後、ダッシュボード（左上のアドレスをクリック）> ネットワーク > Tunnels を選択し、右ペインから「トンネルを作成」ボタンを押下
- 任意のトンネル名を入力し、「トンネルを作成」を押下する。
- インストールと実行欄に表示されている手順に従い作業してください。一番下の「接続状態」に「トンネルが正常に接続されました」と表示されれば完了です。表示される msi をインストールし、コマンドプロンプトを管理者権限で実行、表示されているコマンドを実行するだけです。Ubuntuで環境構築したい場合は Debian の手順で実行してください。完了したら「続行」ボタンを押下してください。
- ダッシュボード（左上のアドレスをクリック）> ネットワーク > Tunnels を選択し、右ペインから作成したトンネル名を選択してください。
- 画面タブの Routes 欄の中にある「+ Add route」ボタンを押下
- 「Published application」を押下
- 表示されたフォームで右上の「Domain」に登録したドメインが表示されていることを確認し、「Service URL*」の欄に `http://127.0.0.1:32287` を入力し、「Add Route」ボタンを押下する。

あとは登録したアドレス（`https://xxxx.f5.si`）にブラウザでアクセスしてみてください。  
`.env` の `NAI_IM_PUBLIC_BASE_URL` の `=` の頭の `#` を削除し、`=` の後ろに上のアドレスを記載してください。起動時にそのアドレスを表示してくれます。  
アカウントの作成やパスワードリセット時にこの値を使用するため間違えないように必ず記載してください。  

## 初期ログイン
起動後に アカウント名とパスワードを入力して作成します。  
マスターアカウント扱いでこのアカウントでしか出来ないことが複数あるので、パスワードを忘れないように気を付けてください。  

## データ保存場所
DB/画像/派生: `server/data/`  
初期化したい場合はこの中のファイル群をすべて削除してください。  
オリジナルファイルもここに保存されます。

## 使い方と機能

### 最初にやること
- 初回起動時はセットアップ画面が開くので、最初のユーザーを作成します。
- ここで作成した最初のユーザーは管理用アカウントとして扱われます。
- 2人目以降のユーザーは、ログイン後のメニューから「アカウント管理」を開き、「作成してURL発行」で登録用URLを発行します。受け取った側はそのURLからパスワードを設定して使い始めます。

### プレビュー管理の使い方
- 上部の「プレビュー管理」では、登録済み画像の検索、閲覧、整理を行います。
- 「作成者」「ソフトウェア」「タグ検索」「日付(開始/終了)」を指定して「検索」を押すと、条件に合う画像だけを一覧表示します。
- タグ検索は、入れたいタグと除外したいタグを条件にできます。
- 「メインプロンプト重複排除」を ON にすると、重複画像をまとめて見たいときの絞り込みに使えます。
- 左側のカレンダーを押すと、その日付で絞り込みます。
- 左側の「作成者（作者）」「ソフト」は、一覧から直接絞り込むためのショートカットです。「解除」を押すとその絞り込みだけ外せます。
- 左側の「ブックマーク」は、自分のブックマークリストや購読中の共有ブックマークから絞り込む場所です。「解除」を押すとブックマーク条件を外します。
- ギャラリー上部の「並び替え」で新しい順、古い順、ブックマーク順に切り替えられます。
- 右上の表示切替で、グリッド表示とリスト表示を切り替えられます。
- 「ページ一括」は今表示しているページだけを選択します。「全対象」は検索結果全体を選択します。
- 「削除」は選択した画像を削除します。通常ユーザーは自分の画像だけ削除できます。
- 画像の星マークはブックマークです。押すとデフォルトのブックマークリストに追加または解除します。
- 画像を開くと詳細画面が開きます。ここでは次の操作ができます。
  - フル表示
  - 元ファイルのダウンロード
  - ポーションファイルのダウンロード（その画像にある場合のみ）
  - タグのコピー
  - 各分類の「全コピー」または「全コピー(強調解除)」
  - メタデータの確認
  - ブックマークの追加・解除
- 詳細画面のタグボタンを押すと、そのタグをコピーできます。
- 詳細画面のブックマーク操作では、どのブックマークリストに入れるかを選べます。新しいリストの作成や、その画像に付いたブックマークの全解除もここから行えます。

### アップロード管理の使い方
- 上部の「アップロード管理」では、画像の取り込みと登録結果の確認を行います。
- 画像は次の方法で取り込めます。
  - 画像ファイルをドラッグ&ドロップする
  - 「ファイルを選択」から複数画像を選ぶ
  - 「フォルダを選択」からフォルダごと選ぶ
  - zip ファイルを「ファイルを選択」から選ぶ
- アップロードが始まると、各ファイルごとの状態と全体の進捗が表示されます。
- 取り込み後は、抽出できたソフトウェア、作成者、タグの集計が右側の「タグアップロードサマリー」に出ます。
- 画像ごとの結果には、成功、重複、失敗などの状態が表示されます。
- zip を送った場合は、送信完了後もサーバー側の処理が続くため、進捗表示で完了まで確認できます。

### 個人設定
- メニューの「個人設定」では、自分の共有設定とブックマークリスト管理を行います。
- 「個人作品の共有」を ON にすると、他のユーザーがあなたを作成者候補として追加できるようになります。
- 「ブックマークの共有」を ON にすると、あなたのブックマークリストを他のユーザーが購読できるようになります。
- 新しいブックマークリストを作成できます。
- 既存のブックマークリストは名前変更できます。
- default のブックマークリストは削除できません。

### アカウント管理（管理用アカウント）
- メニューの「アカウント管理」では、ユーザーの追加と管理を行います。
- 「作成してURL発行」を押すと、新規ユーザー用のパスワード設定URLを発行します。
- 一覧では、各ユーザーの権限、状態、無効化の有無、作成日を確認できます。
- ユーザーを無効化すると、そのユーザーはログインできなくなります。
- 不要なユーザーは削除できます。

### メンテナンス（管理用アカウント）
- メニューの「メンテナンス」では、再解析と統計再集計を実行できます。
- 「再解析（全件）」は、登録済み画像を全件走査してメタデータやタグをもう一度解析し直します。
- 「統計再集計」は、作成者やソフトなどの集計情報を作り直します。
- 実行中は状態、進捗、処理件数、最終更新時刻、heartbeat が表示されます。
- 失敗一覧では、再解析に失敗した画像を確認できます。
- 失敗した画像は「再試行」で再度処理できます。
- 一時的に対象外にしたい画像は「skip」で飛ばせます。再び処理対象に戻したいときは「skip解除」を使います。

### 補足
- メニューの「パスワード変更」は、自分用のパスワード設定URLを開くためのものです。
- ログアウトはメニューから行えます。
- データを初期化したい場合は、上に書かれている `server/data/` 配下のファイルを削除してください。画像とDBも消えます。
